# Formation Recognition Roadmap

## Current Core Modes

- Formation Trainer: study and quiz offensive formations, personnel, and strengths.
- Alignment Mode: study and quiz defensive alignments versus formations.
- Film Mode: read-key clips for quick run/pass, scheme, and direction recognition.
- Pass Concept Mode: chalk-board concept study and quiz for route concepts.
- Blitz Mode: chalk-board blitz install boards with premade pressure and coverage pathways.
- Run Fit Mode: chalk-board mode in progress.

## Future Modes

### Pass Pro ID Mode

Film mode focused on offensive pass protection.

- Upload clips of pass rushes, pressures, and blitzes.
- Users identify who should be blocked based on the protection scheme.
- Future quiz answers could include protection call, hot player, slide direction, and blocked/unblocked threats.

### QB Coverage ID Mode

Film mode focused on quarterback coverage recognition.

- Upload short defensive coverage clips.
- Users identify the coverage from a quick post-snap or late pre-snap look.
- Similar structure to defensive Read Key mode, but built for QBs.

### DL Stunt Mode

Chalk-board mode focused on defensive line stunt recognition and creation.

- Users study and quiz common stunts.
- Users may build stunts by assigning each DL a path.
- Could use the same visual board language as Pass Concept and Blitz Mode.

### Run Fit Mode

Chalk-board mode for defensive run fits.

- Currently in progress.
- The biggest challenge is creating standard pathways that work cleanly across many run schemes and formations.
- We may pivot from “draw every defender’s fit” to a pre-drawn run fit board where users answer one or a few key fits.
- Possible quiz examples: who is force, who is spill, who has cutback, who is the extra fitter, or where does the LB fit versus a specific block.

## Notes

- Film modes should stay clip-based with quick binary or menu answers.
- Chalk-board modes should reuse the board/pathway system where possible.
- Keep study and quiz versions paired whenever the mode benefits from both learning and testing.
